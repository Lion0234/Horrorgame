# HorrorGem Documentation

This directory is the project's durable planning and project-memory space. Keep the active state of the game here so development decisions, current priorities, and known constraints remain easy to recover between work sessions.

## Current documents

- [Game design overview](game-design.md) - high-level game vision, player loop, and mode direction.
- [Progress](progress.md) - completed work, active focus, and immediate next steps.

Add documents as they become useful:

- `architecture.md` - gameplay, technical, asset, and system architecture.
- `roadmap.md` - milestones, delivery order, and near-term priorities.
- `blockers.md` - open questions, dependencies, risks, and decisions needed before work can continue.
- `decisions.md` - concise architectural and design decision records, including rationale.

## Working convention

- Update the relevant document when a meaningful decision, status change, or blocker emerges.
- Give entries a date and keep the current state near the top of each file.
- Mark assumptions clearly, then replace them with confirmed decisions when available.
- Link related documents and Unreal assets or source files where useful.

This is intentionally lightweight: the documents should guide implementation, not become a separate bureaucracy.
